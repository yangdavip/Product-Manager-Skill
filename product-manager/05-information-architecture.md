# 信息架构梳理模块

> 产品经理核心能力之五：信息架构梳理与可视化

---

## 适用场景

- 系统功能模块梳理
- 需求文档的信息架构可视化
- XMind/Mermaid脑图生成
- 项目汇报、需求评审前的架构整理
- 新成员入职的系统认知培训

---

## 分析步骤

### Step 1：识别系统边界

- 系统名称
- 系统定位（服务于谁、解决什么问题）
- 核心业务流程

### Step 2：提取一级模块

- 功能清单表格（FR-001 ~ FR-xxx）
- 目录结构
- ER图中的限界上下文
- 原型文件的页面标题

### Step 3：提取二级功能

- 列表页（查询条件、列表字段、行操作）
- 表单页（新增/编辑字段）
- 详情页（展示区域）

### Step 4：提取三级字段

- 查询条件字段说明表
- 表单字段说明表
- 数据展示字段说明表

### Step 5：提取业务规则

- 状态流转图/状态定义表
- 计算公式/计费模式
- 交互规则说明
- 条件性UI说明

### Step 6：组织层级结构

```
系统
  └── 一级模块（按业务域划分）
       └── 二级功能（按页面/操作划分）
            └── 三级区域（按页面区块划分）
                 └── 四级字段（按具体字段/规则划分）
```

---

## 输出格式

### 格式1：纯文本缩进（导入XMind）

```
系统名称
	一级模块
		二级功能
			三级区域
				四级字段
```

规则：
- 每行以 `- ` 开头
- 子级用Tab缩进
- 无其他符号（#、*等）
- 每行一个节点

### 格式2：Mermaid mindmap（在线渲染）

```mermaid
mindmap
  root((系统名称))
    一级模块
      二级功能
        三级区域
          四级字段
```

## 提取规则

### 从需求规格说明书提取

| 文档位置 | 提取内容 | 对应层级 |
|----------|----------|----------|
| 功能清单表 | 需求ID、需求名称 | 二级功能 |
| 需求详情表 | 需求描述、目标用户 | 二级功能说明 |
| 查询条件字段说明 | 条件项、输入方式、规则 | 四级字段 |
| 表单字段说明 | 字段、输入方式、规则 | 四级字段 |
| 数据展示字段说明 | 字段名称、显示格式 | 四级字段 |
| 行操作按钮说明 | 操作、交互规则 | 三级操作 |
| 状态流转图 | 状态定义、流转规则 | 三级状态 |

### 从原型文件提取

| 文档位置 | 提取内容 | 对应层级 |
|----------|----------|----------|
| 页面注解 | 交互逻辑、状态流转 | 二级功能 |
| 查询条件 | 筛选项 | 三级查询 |
| 表格列 | 列表字段 | 三级列表 |
| 表单字段 | 输入项 | 三级表单 |
| 按钮 | 操作项 | 三级操作 |

### 从技术设计文档提取

| 文档位置 | 提取内容 | 对应层级 |
|----------|----------|----------|
| ER图 | 实体关系 | 数据模型 |
| 表结构 | 字段定义 | 四级字段 |
| 接口设计 | API列表 | 三级接口 |
| 状态图 | 状态流转 | 三级状态 |
| 时序图 | 业务流程 | 二级流程 |
