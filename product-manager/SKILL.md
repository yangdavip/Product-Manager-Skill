# Product Manager Skill

> 专业产品经理核心能力：需求分析、架构设计、需求文档编写、原型设计、信息架构梳理、文档自评审
> 参考标准：CMMI 3.0

---

## 核心原则

### 文档编写四要素

每个需求文档**必须**包含以下四个核心要素：

| 要素 | 说明 | 必填 |
|------|------|------|
| 业务目标 | 解决什么问题？成功标准是什么？ | ✅ |
| 角色权限 | 谁能做什么？权限矩阵是什么？ | ✅ |
| 状态流转 | 业务状态如何变化？触发条件是什么？ | ✅ |
| 用户处理逻辑 | 每个角色在每个状态下能做什么？ | ✅ |

## 工作流程

### 完整文档编写流程

```
需求分析（01模块）
    ↓
业务目标定义（解决什么问题？成功标准？）
    ↓
角色权限定义（谁能做什么？）
    ↓
状态流转定义（业务状态如何变化？）
    ↓
用户处理逻辑（每个角色在每个状态下能做什么？）
    ↓
架构设计（02模块）
    ↓
需求文档编写（03模块，使用SRS/PRD模板）
    ↓
原型设计（04模块）
    ↓
信息架构梳理（05模块）
    ↓
文档自评审（06模块，使用评审检查清单）
    ↓
二次修改
    ↓
最终版本
```

### 需求分析阶段
```
用户调研 → 需求收集 → 需求分析 → 需求评审 → 需求排期
```

### 设计阶段
```
架构设计 → 功能拆解 → 原型设计 → 交互评审 → 设计定稿
```

### 文档阶段
```
PRD/SRS编写 → 自评审 → 二次修改 → 需求评审 → 文档定稿
```

### 开发支持阶段
```
需求答疑 → 设计走查 → 验收测试 → 上线跟踪
```

## 最小化文档集（小型项目）

对于 5人以下、3个月内的小型项目，建议至少维护：
- ✅ 业务目标定义
- ✅ 角色权限矩阵
- ✅ 需求规格说明书（SRS）
- ✅ 原型文件+页面注解
- ✅ 文档自评审记录
