# 原型设计模块

> 产品经理核心能力之四：原型设计与交互规范

---

## 适用场景

- 产品原型设计
- 交互方案评审
- 开发参考依据

---

## 设计框架

### 1. 页面清单

```
页面编号 | 页面名称 | 页面类型 | 所属模块
---------|----------|----------|----------
P-001    | 列表页   | 列表     | 模块A
P-002    | 详情页   | 详情     | 模块A
P-003    | 新增页   | 表单     | 模块A
```

### 2. 页面注解模板

```html
<!-- ================================================
     页面 X 注解：页面名称
     
     【业务目标】
     - 本页面解决什么问题：
     - 成功标准：
     
     【角色权限】
     - 角色1：可见✅ 可操作✅（查看、编辑）
     - 角色2：可见✅ 可操作❌（仅查看）
     - 角色3：可见❌
     
     【交互逻辑】
     - 按钮1：点击后执行XXX
     - 按钮2：点击后执行XXX
     
     【状态流转】
     - 状态A → 状态B：触发条件，操作角色
     
     【各角色处理逻辑】
     - 角色1在状态A：可执行操作1、操作2
     - 角色2在状态A：可执行操作3
     - 角色1在状态B：可执行操作4
     
     【字段规则】
     - 字段1：只读，格式XXX
     - 字段2：必填，校验XXX
     
     【条件性UI】
     - 条件1时：显示XXX
     - 条件2时：隐藏XXX
     
     【数据约束】
     - 约束1：XXX
     - 约束2：XXX
================================================ -->
```

### 3. 原型元素规范

| 元素 | 说明 | 规范 |
|------|------|------|
| 按钮 | 主按钮/次按钮/文字按钮 | 主按钮用于核心操作 |
| 表单 | 输入框/下拉/日期/单选/复选 | 必填项标红* |
| 表格 | 列表/树表/可编辑表 | 支持排序/筛选/分页 |
| 弹窗 | 确认框/抽屉/全屏 | 操作确认用弹窗 |
| 标签 | 状态标签/分类标签 | 不同状态不同颜色 |
| 提示 | 成功/警告/错误/信息 | Toast/Message/Alert |

### 4. 交互规范

| 场景 | 交互方式 |
|------|----------|
| 提交成功 | Toast提示"保存成功"，返回列表页 |
| 提交失败 | 字段下方红色提示，保留已填内容 |
| 删除操作 | 二次确认弹窗 |
| 批量操作 | 先勾选再操作，未勾选提示 |
| 加载状态 | 按钮loading，骨架屏 |
| 空状态 | 显示引导文案/插画 |

## 页面类型模板

### 列表页模板

```html
<!-- ================================================
     列表页注解模板
     
     【交互逻辑】
     - 查询按钮：根据筛选条件查询列表
     - 重置按钮：清空所有搜索条件
     - 新增按钮：跳转到新增页面
     - 批量导出：导出当前查询结果
     - 全选checkbox：勾选/取消当前页所有行
     
     【状态流转】
     - 状态筛选下拉：全部/状态A/状态B/状态C
     
     【字段规则】
     - 列表默认按创建时间倒序
     - 每页默认显示15-20条
     - 鼠标悬停表格行时高亮显示
     
     【条件性UI】
     - 删除按钮：仅当状态=草稿时显示
     - 编辑按钮：仅当状态=草稿时显示
================================================ -->
```

### 表单页模板

```html
<!-- ================================================
     表单页注解模板
     
     【交互逻辑】
     - 保存按钮：提交表单
     - 取消按钮：返回列表页
     
     【字段规则】
     - 必填项以红色*号标识
     - 支持Tab键切换焦点
     - 支持回车键提交
     
     【条件性UI】
     - 字段A选择"是"时，显示字段B
     - 字段A选择"否"时，隐藏字段B
     
     【数据约束】
     - 字段1：最多50个字符
     - 字段2：仅允许数字
================================================ -->
```

### 详情页模板

```html
<!-- ================================================
     详情页注解模板
     
     【交互逻辑】
     - 编辑按钮：跳转到编辑页面
     - 返回按钮：返回列表页
     
     【角色权限】
     - 角色1：可查看所有区域
     - 角色2：仅可查看基本信息
     
     【字段规则】
     - 所有字段只读
     - 金额字段千分位格式
     - 日期字段YYYY-MM-DD格式
     
     【条件性UI】
     - 编辑按钮：仅草稿状态显示
     - 审批按钮：仅待审核状态显示
================================================ -->
```

```
│   ├── form.html           # 新增/编辑页

├── 模块B/
│   ├── list.html
│   ├── form.html
│   └── detail.html
└── common/

    └── script.js           # 公共脚本
```
